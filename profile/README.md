## 👋 Welcome

Agentic workflows are outgrowing the machines that run them. We're building the
substrate that lets fleets of autonomous agents run **safely**, **in isolation**,
and **at scale** — and we're releasing it in the open so the whole field moves
forward.

If you care about running untrusted agents without losing control, or about
coordinating many agents across many machines, you're in the right place. Pull up
a chair.

## 🔬 What we're working on

### ⧉ Agent sandboxing with native tooling per sandbox

We build the isolation layer that lets untrusted agents execute in safe,
inspectable environments — so autonomy never comes at the cost of control. Each
sandbox ships with **native agent tooling built in**: the file, shell, network, and
execution primitives an agent needs to do real work, scoped to the boundary of its
own sandbox. Agents get a full, capable toolkit; the host stays protected.

### △ Distributed agentic infrastructure (A2A across sandboxes)

A single sandbox is the building block. The interesting problems start when you
have many. We're building the layer that lets sandboxed agents **discover, talk to,
and delegate to each other** — agent-to-agent (A2A) communication across isolated
instances — so you can schedule, coordinate, and scale fleets of autonomous workers
reliably and reproducibly, without dissolving the isolation boundaries that keep
them safe.

## 🧭 How we work

- **◇ Open by default.** We release our runtimes, tooling, and findings in the
  open. Agentic infrastructure advances fastest when the foundations are shared.
- **Safe, then fast.** Isolation isn't a feature we bolt on — it's the starting
  assumption everything else is built around.
- **Built to be built on.** Everything we publish, from code to research, is meant
  to be a foundation for others.
